# 通史年表（タイムライン）設計

始皇帝〜溥儀の2166年（前221〜1945年）を1本の横軸に載せ、「統一と分裂の繰り返し」を視覚だけで伝える通史年表ページの設計メモ。[LAYOUT.md](./LAYOUT.md) に散在していた3構想（概要ダッシュボードの「通史タイムライン」・在位データの「在位期間ガントチャート」・王朝横断ビューの「王朝存続期間タイムライン」）を統合する位置づけ。2026-07-20 設計、**同日実装完了**（実装時の設計変更は文末の「実装記録」を参照）。

## 目的（このサイトの文脈での年表の役割）

- **初学者の背骨**: 統計ページはどれも「王朝」「時代」を軸に集計しているが、その時間的な並びを一望できるページがない。年表は「サイト内のどの統計もこの軸の上にある」と理解するための共通基盤になる。
- **統一⇄分裂のリズムの可視化**: 並立王朝を縦にレーン積みすると、三国・五胡十六国・南北朝・五代十国で帯が縦に膨らみ、統一期は1本に収束する。説明文なしで中国史の大構造が伝わる。
- **既存統計へのハブ**: 皇帝セグメントのクリックで共有詳細ダイアログ（全12項目＋全指標順位）を開き、年表→個人→各統計ページの導線を作る。

## ページ位置・ナビ

- 新設ページ **`/timeline`**（ページ名「通史年表」）。
- ナビはトップレベル項目として「概要ダッシュボード」の直下に置く（カテゴリ配下に入れない。年表は特定の統計項目に属さない横断ビューのため）。
- トップページのセクションカードにも1枚追加する。

## 情報構造（3層）

```
┌─ ミニマップ（全期間・常時表示・ナビ兼用） ─────────────────────────┐
│ [秦・前漢][新〜後漢初][後漢][三国][晋][五胡十六国/南北朝][隋][唐]…[清] │ ← 時代帯
│ ▁▁▂▁▁▃▅▂▁▁▅█▃▁▂▅▂▁▁▁                                │ ← 同時在位皇帝数カーブ
│         └──┘ ← 現在のビューポート窓（ドラッグ/クリックで移動）        │
└─────────────────────────────────────────────────┘
┌─ 本体（横スクロール・離散ズーム3段） ────────────────────────────┐
│ 100年目盛 ──┬────────┬────────┬──                        │
│ 魏   ████│███│██     ← 王朝帯の中に皇帝ごとのセグメント          │
│ 蜀漢  ██████│███                                          │
│ 呉   ██│████│█│██    ← 並立王朝は縦にレーン積み               │
│ ░░░░ ← 空位期間は斜線ハッチ＋注記                              │
└─────────────────────────────────────────────────┘
```

1. **時代帯**: `ERA_BY_SECTION` の時代区分ラベル（秦・前漢〜清の約20区分）。他ページと語彙を完全に揃える。本体側では上部にスティッキー表示し、深くズームしても現在の時代を見失わないようにする。
2. **王朝帯**: `dynasty.name`+`section` 複合キー（87王朝）を1本の帯として、並立をレーンに積む。
3. **皇帝セグメント**: 王朝帯の内部を `reigns[]` ごとに区切る（365人・復位者は複数セグメント）。ホバーで `EmperorTooltip`、クリックで共有詳細ダイアログ。

## データ設計（すべてビルド時計算）

`site/src/lib/emperors.ts` に `getTimelineData()` を追加（Server Component 専用、fs 依存側）。Client へはシリアライズ済みの配置データのみ渡す（既存の分離方針を踏襲）。

```ts
type TimelineData = {
  eras: { label: string; startYear: number; endYear: number }[];
  dynasties: {
    key: string;            // name+section 複合キー
    label: string;          // 表示名（同名王朝は時代付きで区別、既存の重複回避ロジックを再利用）
    category: "正統" | "割拠" | "反乱・自称"; // dynasty.category から導出
    lane: number;           // レーン番号（0 = 最上段）
    colorSlot: number;      // --series-N 割当
    spans: { startYear: number; endYear: number }[]; // 在位カバレッジの合併区間（複数 = 帯内ギャップあり）
    segments: {
      emperorId: string;
      label: string;        // 表示名（commonName null フォールバック込み、既存ロジック再利用）
      startYear: number; endYear: number;
      isRestoration: boolean;
    }[];
  }[];
  vacancies: { startYear: number; endYear: number; label: string }[]; // 全王朝共通の空位期間
  concurrency: { year: number; count: number }[]; // 年ごとの同時在位皇帝数（ミニマップ用）
};
```

### 王朝帯 = 「収録皇帝の在位カバレッジの合併」（min-max スパンではない）

帯は所属皇帝の `reigns[].startYear/endYear` を合併（union）した区間の集合として描く。**王朝の「建国〜滅亡」を別途調査して描くのではない**（データ正確性の原則: 新規の歴史判断を持ち込まず、収録済みデータの純粋な写像にする）。この方式が正しく効く実例（2026-07-20 実データで確認済み）:

- **帯の内部にギャップを持つ王朝が7つある**: 後趙（335–348 = 石虎の天王期）・前秦（358–384 = **苻堅の天王期**。苻堅は皇帝を称していないため収録対象外）・後秦・唐（691–704 = 武周への中断）・閩・西遼・清（1913–16と1918–33の中断、1917 = 張勲復辟、1934–45 = 満洲国）。
- 帯内ギャップは**点線コネクタ**（淡墨1px）で同一王朝の連続性を示しつつ、皇帝がいた期間だけを実帯にする。ホバーで「この期間、○○は天王を称し皇帝ではなかった」等の説明が出せるとなお良い（文言は収録基準ドキュメントの範囲内で書ける。原典の追加調査は不要）。

### 空位期間（全王朝共通のグローバルギャップ）

実データ上、皇帝が1人もいない年は4区間（2026-07-20 集計）:

| 期間 | ラベル文言案 |
|---|---|
| 前206〜前203 | 皇帝不在（秦滅亡〜漢建国。項羽は皇帝を称さず） |
| 8年 ※ | 皇帝不在（王莽の居摂期。孺子嬰は即位せず） |
| 1913〜1914 | 皇帝不在（中華民国） |
| 1918〜1933 | 皇帝不在（中華民国。1917年に張勲復辟、1934年に満洲国帝政） |

※機械集計上は「7年」だが、居摂は6〜8年のため表示文言は実装時に前後の在位端と突き合わせて確定する。全レーンの高さを貫く斜線ハッチ帯として描き、「皇帝を名乗った人物のみ収録」という収録基準の可視化として積極的に見せる。

### 座標系（紀元前の扱い）

- データの年は「前221年 = -221」（0年なし・天文学的紀年ではない）。**内部座標は `t = year < 0 ? year + 1 : year` で連続化**して配置計算し（-1年と1年の間が1年になる）、表示ラベルだけ「前n年」に戻す。x座標変換はこの1関数に集約する。
- 年単位で描く（`startDate` の日付精度は使わない。2166年スケールでは1年未満の差は視認できず、日付不明の皇帝も多いため年で統一する方が正直な表現）。

### レーン割当

- 開始年順・**正統優先**（`category` が「正統」→ 反乱・自称/割拠の順）の貪欲法（interval partitioning）。正統王朝が上のレーンに来て、割拠政権が下にぶら下がる。
- 実測で**全期間9レーンに収まる**（並立王朝数の最大は619年の9。隋末群雄期）。レーン数は固定せず算出結果を使うが、高さ設計は9レーン想定で足りる。
- 王朝の帯内ギャップ区間ではそのレーンを他王朝に貸さない（点線コネクタが横切られると読めなくなるため。このルールでも必要レーン数は9のまま — 2026-07-20 実測）。

### 並立数カーブ

年ごとの同時在位皇帝数（ピークは618年の14人）。ミニマップに淡い面グラフとして重ね、「帯が厚い時代 = 分裂期」の直感を数値で裏付ける。「統一期/分裂期」の**手動分類はしない**（編集判断の持ち込みになるため。カーブとレーン積みだけで伝わる）。

## ズーム・ナビゲーションUI

連続ズーム（ホイール等）は実装しない。ページスクロールとの競合・実装複雑度・モバイル対応の問題があり、離散3段で十分なため:

| ズーム | スケール | 全幅の目安 | 表示内容 |
|---|---|---|---|
| 全体 | コンテナ幅フィット（約0.55px/年） | 1画面 | 時代帯＋王朝帯のみ（皇帝区切りなし） |
| 標準 | 2px/年 | 約4,300px | ＋皇帝セグメント区切り（ラベルなし） |
| 詳細 | 8px/年 | 約17,300px | ＋皇帝名ラベル（帯高に収まる範囲）・復位マーカー |

- ズーム切替時は**画面中央の年を維持**してスクロール位置を再計算する。
- ミニマップは常時全期間表示で、現在のビューポート窓を矩形表示。クリック/ドラッグでジャンプ。時代帯ラベルのクリックでその時代の先頭へスクロール。
- 軸目盛はズームに応じて100年/50年/10年刻み。
- モバイルは同じ離散ズーム＋横スクロールで対応（ピンチズームは実装しない）。

## 皇帝セグメントの操作性（短在位対策）

在位1年未満の皇帝が**65人**おり、詳細ズーム（8px/年）でもセグメント幅は8px未満、完顔承麟（在位1日未満）は事実上幅0になる。バー矩形そのものにヒットさせる方式は既存ランキングチャートで破綻済み（`RowOverlay` の教訓）。年表では:

- **帯単位の透明オーバーレイでポインタを受け、x座標から最近傍セグメントにスナップ**する（帯内の mousemove → 二分探索で「その位置に最も近い皇帝」を特定）。帯の上なら必ず誰かにヒットするため、幅0のセグメントも選べる。ホバー中のセグメントは薄いハイライト＋ツールチップ、クリックで詳細ダイアログ。
- キーボード: 帯にフォーカス → 左右キーで皇帝を前後移動・Enter でダイアログ（roving tabindex）。ただし主たるa11y導線は下記「表で見る」とする。

## 見た目（水墨文人テーマとの整合）

- **王朝帯の配色**: 検証済み8色パレット（`--series-1〜8`）を「同時期に重なる王朝・同一レーンで隣接する王朝が同色にならない」制約でビルド時に巡回割当。最大並立9 > 8色だが、重なり時の同色は正統/割拠の表現差（下記）で区別できる。実装時に dataviz skill の手順で surface `#F5F1E8` に対して再検証すること。**未確定のまま残っている `--chart-1〜5`（王朝パレット）はこの実装で確定させる**（LAYOUT.md の申し送り事項）。
- **正統/割拠の視覚差**: 正統王朝 = 帯高さ大（例: 28px）・不透明。割拠（category「割拠」）・反乱自称 = 帯高さ小（例: 18px）・同色相の淡色（例: 55%不透明度）。初学者が「本流」を見失わないための最重要の differentiation。数値は実装時に調整。
- **皇帝区切り**: セグメント間は surface 色1pxの区切り線（既存の死因積み上げチャートと同じ手法）。
- **復位**（`isRestoration`）: セグメント左端に朱の小マーカー（▲等）。朱は「ここぞ」のワンポイントに限る方針と整合。
- **空位期間**: 淡墨の斜線ハッチ（CSS `repeating-linear-gradient`、朱は使わない）＋縦書きまたは回転ラベル。
- **帯内ギャップ**: 点線1px（墨色30%）。
- **時代帯**: 生成り（`--sidebar` #ede7d8）と宣紙色の交互＋明朝体ラベル。

## パフォーマンス設計（既存教訓の踏襲）

- **SVG（Nivo）は使わない**。Nivo にガント/タイムライン型はなく、div 絶対配置＋`transform: translateX()` の自前実装とする。layout-shift 計上回避のため位置指定は必ず transform（`top`/`left` 書き換え禁止 — timespan 計測第1・2弾の教訓）。
- **横方向ウィンドウイング**: `useWindowedRows` の横版 `useWindowedSpan` を新設。可視範囲±オーバースキャンのセグメントだけ DOM に載せ、**量子化ステップ**（例: 256px）の境界をまたいだときだけ state 更新（毎フレーム setState 禁止の教訓）。要素の key はウィンドウ内 index ではなく王朝キー/皇帝 id（RowOverlay の index-key は「行位置固定・中身差し替え」が理由だった。年表は要素位置が id に固定なので id-key が正しい）。
- **ホバー/ダイアログ状態はチャート本体に持たない**: `useTipOutlet` / `useDetailOutlet` をそのまま使う。mousemove のスナップ計算は二分探索のみで、チャート再レンダリングを発生させない。
- **スクロール直後のホバー抑制**（`hoverAllowed()`）も踏襲。
- ページ下部のコンテンツ（表で見る等）は `LazyMount`。
- ズーム切替の再レイアウトは O(87帯+365人) の配置計算のみで軽量。切替直後500ms超のシフトが CLS に化ける教訓があるため、切替時は全高を先に確保してから中身を差し替える。

## データ正確性上の注記（/about に追記する文言の骨子）

1. 帯は「収録皇帝（皇帝号を称した365人）の在位期間」の写像であり、王朝の建国〜滅亡年とは一致しない場合がある（例: 清の帯は1636年から。後金ハン時代は含まない）。
2. 帯の途切れ・空白は「皇帝を名乗った人物がいない期間」を意味する（例: 前秦の358〜384年 — 苻堅は天王を称し皇帝ではない。収録基準参照）。
3. 年単位で描画しており1年未満の在位は幅を持たないことがある（クリックでの参照は可能）。

## 実装記録（2026-07-20）

第1〜3弾を一括で実装した（`site/src/app/timeline/page.tsx`・`site/src/components/timeline/timeline-explorer.tsx`・`timeline-table.tsx`・`site/src/lib/emperors.ts` の `getTimelineData()`）。`tsc`・`eslint`・`next build` 通過、headless Chrome 実操作（ズーム3段・fitクリック拡大・ホバー・ダイアログ・ミニマップジャンプ・表・モバイル390px）で検証済み・コンソールエラー0件。実装時に設計から変更・具体化した点:

- **横方向ウィンドウイングは不要だった**: 描画は Nivo でなく div 絶対配置の自前実装で、総ノード数は帯87＋セグメント365程度と小さい。ネイティブ横スクロールでは React の再レンダリングが一切走らないため、`useWindowedSpan` は作らなかった（ランキングチャートのウィンドウイングは「Nivo の SVG 364行」が高コストだったのが理由で、前提が違う）。
- **レーンは正統/非正統の2ブロック分離に変更**: 設計の「開始年順・正統優先の貪欲法」だけでは、唐・明などの本流が割拠政権に挟まれて下のレーンに落ちた。正統王朝ブロック（5レーン）と並立・反乱政権ブロック（5レーン、間に `GROUP_GAP`＋点線区切り）に分けて別々に詰めることで、**lane 0 に「秦→前漢→…→北宋→南宋→明」の本流がほぼ一列に連なる**。同年の禅譲交代（後漢→魏の220年など）は同一レーンを許容（`astroYear(start) >= astroYear(end)`）。計10レーン。
- **時代帯レーンは優先割当**: 開始年順の貪欲法だと隋（〜619年の皇泰主）と隋末に挟まれた唐が3段目に落ちる。移行期・並立期（新〜後漢初・五胡十六国・隋末）を後回しにする優先順で区間の空きに差し込む方式にし、主要時代が上2段・並立期が3段目に収まる（計3レーン）。
- **fitズームのクリックは「その地点へ拡大」**: 全体ズームではセグメントを描いていないため、クリックで皇帝ダイアログでなく拡大ズームへ遷移（ホバーも王朝サマリーのツールチップ）。全体→時代→皇帝と段階的に潜る初学者導線になった。
- **配色は淡彩置き**: `--series-1〜8` をそのまま塗らず `color-mix` で宣紙色に混ぜた淡彩（正統40%/並立22%）＋同色のエッジ線。水墨文人の地色に馴染み、帯上の墨色ラベル（宣紙色のハロー付き）のコントラストも確保できる。スロットは「同時期の重なり＋同一レーンの隣接」を避けて割当。**`--chart-1〜5` の確定はしていない**（今回も王朝ごとの固有色は使わない構成になったため。申し送り継続）。
- **座標系の教訓**: ズーム中心維持の `pendingCenterRef` に「絶対astro年」と「t0からのオフセット」が混在し、fitクリック拡大が約200年ずれるバグを実機検証で発見・修正した。**px↔年の変換を跨ぐ値は必ず絶対astro年で持ち回す**こと。
- 軸目盛はズーム倍率から `step×pxy>=36px` を満たす最小刻み（10/50/100/250/500年）を選ぶ（スマホの全体ズームでラベルが重なるため）。
- 「見方の注意」は /about でなく /timeline ページ内のセクションに置いた（チャートの直下にある方が読まれる。/about へは区分・収録基準のリンクで誘導）。
- ミニマップの表示窓は全体ズーム時（窓＝全幅）は非表示。窓の同期・スクロール追従は React state でなく DOM 直接操作（transform）。

### キーボードナビ（2026-07-20 追補実装）

第3弾で将来案としていたキーボードナビを同日中に実装した。設計時案の「帯ごとの roving tabindex」でなく、**スクロールコンテナ1個をフォーカス可能（`role="application"`）にして矢印キーで在位セグメントを移動する方式**（tabストップが87個増えるのを避ける）。左右＝同じレーンの皇帝を時代順に移動（帯の端では同レーンの前後の帯へ。lane 0なら本流を通しで辿れる）、上下＝同時期に並立する最も近いレーンの王朝へ（同時期の帯が無ければ移動しない）、Enter/Space＝全体ズームではその地点へ拡大・それ以外は詳細ダイアログ（クリックと同じ段階導線）、Escape＝解除。フォーカスは `{bandIdx, segIdx}` で保持し朱色リングを描画（座標はズームごとに再計算）、移動時は画面内スクロール＋ホバーと同じツールチップ表示＋`aria-live` で皇帝名を読み上げ。フォーカス移動のプログラムスクロールが既存の「スクロールで tip を消す」処理に拾われるため、`suppressTipClearRef` で1回分抑制する。「表で見る」は引き続き代替導線。

## 第2世代「大河ビュー」（2026-07-21 全面再設計・実装）

第1世代（上記）はユーザーレビューで「許容できるクオリティではない。既存の成果物にこだわらず世界一わかりやすいタイムラインを目指して」と差し戻され、マクロ構造の分かりやすさを最優先に全面再設計した。モック（`mockups/timeline-river.html`・生成スクリプト `timeline-river.gen.py`。実データ埋め込みのインタラクティブHTML）で方向性の承認を得て本実装した。第1世代からの根本的な変更:

- **87帯の全表示をやめ、約35本の「流れ」+群雄クラスターに集約**: 五胡十六国・隋末群雄・十国・元末群雄などの小政権は灰ハッチの「まとまり帯」1本に集約し、**クリックで構成政権のミニレーンに開閉**できる（開いた行だけ縦に広がる）。集約は `site/src/lib/timeline-river.ts` の `STREAM_DEFS`（キュレーション表）で定義し、**全 dynastyKey の被覆をビルド時にassertする**（新規収録時に表の更新漏れがあるとビルドが落ちる）。
- **縦軸に地理の意味**: row 0=中央=天下統一の座、負=北方（北魏〜北周・遼・金・西夏・北元・入関前の清）、正=南方（蜀漢・呉・東晋・南朝・十国・南宋・南明）。第1世代の「正統/非正統の2ブロック+意味不明と指摘された点線区切り」を廃止。清の入関（-1→0）・元の北走（0→-1）は帯自体がS字の流路で段替わりする（`splits`/`rows` 指定でスパンを切断）。
- **帯の太さ=「唯一在位」**: その年に在位する皇帝がこの流れの王朝にしかいない期間（=統一）は帯が太くなる（34px/17px）。在位データからの機械算出で、歴史的な領土統一とは異なる境界事例（東晋371–383・南宋1235–1259）はページ内「見方」セクションに明記。ただし**1912年（宣統帝退位・帝制終焉）より後は打ち切り**（後述の追補修正）。下部に同じ判定の**統一/分裂バーコード**を敷く。
- **継承・分岐・吸収のテーパー**: 禅譲/継承(succ)・分岐(fork)・吸収(merge)・空白期を挟む継承(dash)の4種を `EDGE_DEFS` で列挙し、帯の下層に描く（後漢→魏、魏蜀呉→西晋、北周→隋、五代→北宋など39本）。
- **章（時代）ゾーン+画期フラグ+時代ジャンプ**: 時代帯レーンの代わりに章ゾーン（交互の淡背景+ラベル）と16の画期注記（朱のフラグ。データ項目ではなくUI上の注記）。ミニマップは廃止し、ズームボタン横の**時代ジャンプチップ**（秦・漢〜清）で代替。
- **ズーム時のラベルはスクロール追従**: 拡大/詳細ではストリームラベルをスパン左端に置き、スクロール時に `data-clamp-min/max` 属性を rAF+`transform` で更新してスパン内にクランプする（第1世代同様、スクロールでの React 再レンダリングはゼロ。CLS対策で top/left は書き換えない）。
- **全体ズームに下限1.05px/年**: コンテナ幅フィットだけだと注記・ラベルが衝突するため、モックと同じ密度を下限にし、狭い画面では横スクロールにする。
- **配色は意味ベースの系統色**: 漢系=金(series-4)・晋系=紫(7)・北族=青(1)・宋=緑(2)・明=朱(8)・隋=青緑(5)など、`color-mix` 淡彩置きは第1世代を踏襲。橙(6)と桃(3)は時空間で隣接させない（dataviz検証の Normal-vision floor 対策。帯は全てラベル+輪郭線の二次符号化あり）。
- **踏襲したもの**: `useTipOutlet`/`useDetailOutlet` によるホバー/ダイアログ状態の分離、スクロール直後150msのホバー抑制、`role="application"` のキーボードナビ（左右=同じ段・上下=並立・Enter=拡大/詳細/クラスター開閉・`aria-live` 読み上げ。操作単位は非クラスター=在位セグメント/クラスター=帯）、「表で見る」（`timeline-table.tsx` は旧 `getTimelineData()` のまま無変更）、皇帝クリック→詳細ダイアログ。
- **実装ファイル**: `lib/timeline-river.ts`（データ構築。年は全て天文年）+ `components/timeline/river-timeline.tsx`（SVG描画。第1世代の div 絶対配置から SVG に変更 — テーパー曲線・ハッチパターンのため）。旧 `timeline-explorer.tsx` は削除。`getTimelineData()` は表ビュー用に存続。
- **検証**: tsc/eslint/next build 通過、実ブラウザで全体/詳細ズーム・清へのジャンプ・五胡十六国の開閉・皇帝ツールチップ・康熙帝ダイアログを確認、コンソールエラー0件（スパン分割をまたぐ在位セグメント（順治帝・元恵宗）の key 重複と描画はみ出しを実機検証で発見・クランプ修正した）。

### 自己レビュー修正（2026-07-21 同日）

実装直後の自己レビューで検出した問題を修正した:

- **人数表記の手書き禁止**: STREAM_DEFS に手書きしていた補足の皇帝数が実データと不一致（唐21↔実24人 — 唐を自称した反乱・自称3人が同じ dynastyKey に合流するため。五代13↔実14人）、さらにツールチップで機械算出値と連結され「皇帝14人・皇帝14人」と二重表示になっていた。**人数はラベル・ツールチップとも `emperorCount`（クラスターの政権数は `members.length`）から導出**に変更し、sub は「5王朝」等の補足のみ残した。
- **展開クラスターの配置**: 展開ボックスを行の中央でなく**脊柱（row 0）の反対側**に足す方式に変更（北側の行は上へ、南側は下へ拡張。通常帯は据え置き）。同じ段の他の帯（五胡十六国展開時の北魏など）とボックスが重ならなくなった。
- **交互の濃淡を太さ境界で分割**: 唯一在位⇄並立で帯高が変わる位置をまたぐ在位の濃淡矩形を、sole ピースとの交差ごとに帯の実高さで分割描画。
- **詳細ズームの皇帝名もクランプ**: 長い在位（康熙帝61年など）の名前にストリームラベルと同じ data-clamp 機構を適用し、スクロール中もセグメント内で見え続ける。
- キーボードの微修正: 左右キーで隣のストリームへ渡るとき同じ段のスパン内に着地する（清のように段をまたぐストリームで別の段へ飛ばない）。programmatic スクロールの suppress フラグはスクロール可能域へのクランプ後に比較（クランプで同値になるとフラグが残るため）。
- ドキュメント: site/AGENTS.md の通史年表の記述を第2世代へ更新。
- 実機再検証: ツールチップ二重表示の解消・唐の皇帝24人表記・展開ボックスの分離・キーボード（左右/上下/Enter開閉・フォーカスリング）・拡大ズーム（展開状態の維持・ラベルクランプ）・モバイル400px幅（横スクロールで破綻なし）を確認。tsc/eslint/build 通過。

### 帝制終焉後は「唯一在位」を打ち切り（2026-07-21 追補・ユーザー指摘）

当初の機械判定では帝制終焉後の洪憲1915–16・張勲復辟1917・満洲国1934–45も「在位皇帝が単一王朝のみ」に該当し太い帯（統一）になっていたが、共和制下の帝政であり天下統一とは呼べないため、**1912年（宣統帝退位）より後は単一王朝でも sole=false に打ち切る**よう変更した（`timeline-river.ts` の `SOLE_END` 定数。清の帯は退位年の1912年までは太いまま）。帯の太さと統一/分裂バーコードの両方に適用し、ページ脚注・バーコード凡例の文言も更新。判定はあくまで機械算出+単一の明示的カットオフであり、期間ごとの手動分類はしていない。
