- 層A/B の検証スクリプト（verify_quotes*.py / verify_calendar.py）は再実行可能。CI 恒久化する場合は上記正規化の改善とセットで

## 残る限界（これでも「完全」ではない）

0. **98.6% は「実在の上限」**: 照合は6〜10字断片＋字間ギャップ許容のため、「大赦天下」等の頻出句は偶然一致し得る。逐語完全一致の保証ではない。また**存在≠正しい帰属**（sui-yanghao 型: 引用は実在するが別書・別事象由来）は存在チェックの構造的な取りこぼし
1. 解釈層（引用→カテゴリ判定・計上判断・数え年規約）は判断であり機械照合不能。既存の個別調査プロセス＋過去QA（Wikidata 照合等）が担保
2. コーパス自体の欠落・版本差（ming-huizong の例、四庫本 vs 百衲本）
3. コーパス外資料57件（実録・近現代資料・満洲国公報等）は照合対象外
4. 一次史料同士の矛盾（chen-feidi 型）は「どちらが正」を決められない

→ さらに上を狙う場合は層C（値の意味的再導出: 各人物のキャッシュ原文から時年・享年・カテゴリを独立再導出して突合）。試算: 365人×主要フィールドで数千万トークン規模・要 Workflow。
