# 死因スキーマ設計（`deathCause`）

`data/emperors.json` の各人物レコードに追加する死因フィールドの設計ドキュメント。
在位データ（`meta.status.phases.reignData`）完了後の追加調査項目のうち、優先度1位（`meta.status.phases.deathCause`）に対応する。

調査は完了済み（`meta.status.phases.deathCause.status` は `"completed"`、364人全員分確定、2026-07-16）。完了ブロックは `meta.deathCauseCompletedBlocks` を参照。

## 情報源（原典優先）

判定の根拠は必ず一次史料（正史の本紀・列伝）の原文に置く。手順の詳細は CLAUDE.md の「データ調査の進め方」を参照。要点：

- **第一選択**はローカルに取得済みの正史コーパス（`china-history/`・`daizhigev20/`）で該当正史原文を確認し、死因に関する記述を確認する。ローカルで確定できない場合のみ、WebFetch で維基文庫（`zh.wikisource.org`）にアクセスする（サーバー負荷に配慮し必要な篇のみ）。詳細は [docs/process/RESEARCH_PROCESS.md](../../docs/process/RESEARCH_PROCESS.md) を参照。
- 原文に到達できない場合のみ WebSearch を使うが、**検索結果の要約をそのまま採用せず、原文の直接引用（漢文）を含む結果を探して採用する**。個人ブログ・まとめサイトの解説文だけを根拠にしない。
- 2026-07-15、秦・前漢・玄漢・後漢32人分の死因判定がWebSearchの要約のみを根拠にしていたことが判明し、Workflowで原典を1人ずつ再検証した。5件で誤り・過大なconfidenceが見つかった（例：後漢・沖帝＝次代の質帝の記述との混同で「梁冀弑逆説」を誤採用していた。原典には沖帝の死因記述自体がなく `不詳` に修正）。**WebSearchの要約だけで判定を確定させない**という教訓をここに残す。

## フィールド構造

各人物レコードに `deathCause` オブジェクトを追加する。

```json
"deathCause": {
  "category": "暗殺",
  "note": "自然文での説明（誰に・どのような経緯で）",
  "confidence": "high",
  "source": { "page": "...", "lang": "ja" }
}
```

- `category`: 下記カテゴリ一覧から単一値（enum）を選ぶ。
- `note`: 経緯を自然文で記述。手段（毒殺・斬首など）や複合的な経緯（例：敗戦→追い詰められて自尽）はここに書く。
- `confidence`: `high` / `medium` / `low`。情報源間で不一致が残る場合は `medium` にし、経緯は `verification.notes` にも残す（既存の `verification.confidence` と同じ運用）。
- `source`: 参照ページ・言語（既存 `reigns[].duration.source` と同形式）。

## カテゴリ一覧（`category`）

| 値 | 定義 |
|---|---|
| `病死` | 自然死・疾病による死（老衰含む） |
| `暗殺` | 同一政権内の臣下・近親・宦官等による謀殺。毒殺（鴆殺など）もここに含める |
| `処刑` | 敵対勢力・後継政権・征服者による裁判・見せしめ的な公的処断（例：亡国後に征服者に処刑された場合） |
| `戦死` | 親征・防衛戦・鎮圧戦などの戦闘中の死 |
| `自尽` | 自殺・自害（廃位や敗戦に追い詰められての自裁を含む） |
| `事故死` | 落馬・溺死・火災等の事故性の死 |
| `不詳` | 死因の記録・手がかりが原典に見当たらない |
| `諸説あり` | 複数の原典/通説が対立し一つに絞れない場合 |

### 設計判断（確定事項）

- **暗殺と処刑の境界は「誰が」実行したかで区別する**。同一政権内部の者（臣下・皇族・宦官）による謀殺は `暗殺`、政権交代後の勝者・征服者による公的な処断は `処刑`。判断が微妙なケース（例：亡国の君主が降伏後に幽閉先で殺害された場合）はケースバイケースで原典の書きぶり（「殺した」か「処刑した/斬した」か）を見て判定し、迷う場合は `verification.notes` に判断根拠を残す。
- **臣下等に自裁を強要された場合は、実行の形式（本人の手による）を優先し `自尽` に分類する**（ユーザー判断、2026-07-15 秦・二世皇帝で確定）。実質的な首謀者がいる場合も、`note` に経緯（誰の強要か）を明記する。「暗殺」は加害者が直接手を下した謀殺に限定する。
- **毒殺は独立カテゴリにせず `暗殺` に含める**。手段の詳細は `note` に明記する（例：「毒殺（鴆殺）」）。
- **カテゴリは単一値**。複数の死因要素が絡む場合（例：戦いに敗れた末の自尽）も主要因ひとつを選び、経緯の複雑さは `note` の自然文で表現する。配列化や主/副の2段構えは採用しない。
- **`不詳` と `諸説あり` は分ける**。`不詳` は記録が全くない場合、`諸説あり` は複数の原典/通説が対立し一つに絞れない場合。`諸説あり` の場合は `note` に対立する説を併記し、`verification.notes` にも経緯を残す。

### 既存方針との整合

- 判定の一次的な根拠は原典（正史）に置く（CLAUDE.md「原典優先・でっち上げ禁止」方針）。ただし**史書自体が恣意的に史実を書き換えている可能性がある**（易姓革命後の正史編纂では、前王朝最後の皇帝が実際以上に悪く描かれる、勝者側の行為が正当化される等の偏りが知られる）ため、原典の記述を鵜呑みにせず、必要に応じて現代の学術的な通説・研究も調査する（ユーザー指示、2026-07-15）。
  - 原典の記述と現代の学術的通説が一致する場合は、そのままcategory・noteに反映する。
  - 対立する場合は、原典の記述をベースにしつつ、学術的に有力な異説を`note`（または`諸説あり`カテゴリ）に併記する。単なる俗説・二次創作（小説・ドラマ由来の逸話等）は学術的通説とみなさず採用しない。
  - どちらを採用するか迷うケースは都度ユーザーに確認する。
- 現代の科学的検証（法医学調査・考古学的出土品調査等の客観的物証）がある場合は例外的に重視してよいが、都度ユーザーに確認してから採用する。

## 調査時の参考データ（既存の手がかり）

在位データ調査時点で、`reigns[].note` にすでに死因の手がかりが自然文で残っている人物が364人中134人存在する（暗殺・処刑・病死・自殺等のキーワードを含むもの）。死因調査（グループ1）はこれらの再確認から着手すると効率的。
